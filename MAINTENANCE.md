# 文档维护指南

这份指南用于后续新增、整理和复盘资料，目标是保持资料可检索、可追溯，并尽量不破坏历史记录。

## 基本原则

- 原始记录优先保留，不直接覆盖旧内容。
- 索引文档只做导航和摘要，不承载大量正文。
- 图片与引用它的 Markdown 放在同一目录，避免相对路径失效。
- 新增文档后同步更新对应目录的 `README.md` 和根目录 [CATALOG.md](CATALOG.md)。

## 命名规范

| 类型 | 推荐命名 | 示例 |
|---|---|---|
| 周报 | `YYYY-MM-DD.md` | `2024-05-24.md` |
| 面试日期笔记 | `Interview/social/YYYYMMDD/YYYY-MM-DD.md` | `Interview/social/20240524/2024-05-24.md` |
| 招聘记录 | `YYYY春招.md`、`YYYY秋招.md` | `2024春招.md` |
| IDP | `YYYYIDP.docx` 或 `YYYYIDP.md` | `2024IDP.md` |
| 生活计划 | `YYYY-MM主题.md` | `2024-06美食月.md` |

历史文件可以保持原命名，不需要为了统一格式而移动。

## 新增文档检查清单

1. 文件放入对应一级目录。
2. 日期类资料按时间命名，必要时单独建日期目录。
3. 如果文档内引用图片，确认图片与 Markdown 的相对路径可以打开。
4. 更新对应目录 `README.md`。
5. 更新 [CATALOG.md](CATALOG.md) 的文件索引或主题索引。
6. 提交前运行一次 Markdown 链接检查。

## 复盘建议

| 目录 | 建议复盘方式 |
|---|---|
| `Interview/` | 按业务设计、技术基础、项目经验、系统排查四类提炼高频题。 |
| `Recruitment/` | 按背景、目标、渠道、过程、结果、经验教训沉淀活动复盘。 |
| `WeeklyReport/` | 每月或每季度提炼一次项目进展、关键产出和遗留问题。 |
| `IDP/` | 每半年对照目标和行动计划做一次进展复盘。 |
| `Food/` | 适合保留为轻量生活清单，不必过度结构化。 |
