# 2021 春招直播复盘

来源：[2021春招.md](2021春招.md)

本文从春招直播前序交流记录中提炼可复用的招聘表达、岗位卖点和复盘要点。

## 直播定位

| 项目 | 内容 |
|---|---|
| 形式 | 30 分钟左右的轻量直播，偏岗位推介和互动表达。 |
| 风格 | 类似带节奏的直播推介，需要语速较快、有激情、表达清晰。 |
| 平台 | Boss 直聘平台，预计流量高于上次 AUC 平台活动。 |
| 露出 | 后台岗位介绍约 10-15 分钟。 |
| 重点 | 不重点讲完整流程和公司 PPT，而是快速向候选人推介岗位吸引力。 |

## 后台开发岗位卖点

| 卖点 | 可讲述角度 |
|---|---|
| 核心程度 | 后台开发承担核心业务能力，能参与关键链路建设。 |
| 成就感 | 技术方案能直接影响业务效率、稳定性和用户体验。 |
| 发展空间 | 技术公司、全领域机会、技术驱动，适合持续成长。 |
| 技术栈 | 以 Java 为主，但不对语言做硬性限制，可入职后培养。 |
| 学习氛围 | 技术分享、开源项目、个人学习和团队成长空间。 |
| 机会规模 | HC 多，机会大，适合积极投递和内推。 |

## 可复用话术框架

### 1. 岗位负责什么

- 后台开发负责核心业务系统和服务能力建设。
- 工作内容不仅是写接口，也包括稳定性、性能、业务抽象和工程效率。
- 需要理解业务，同时用技术方案支撑业务快速迭代。

### 2. 为什么值得来

- 岗位足够核心，有机会接触真实大流量和复杂业务问题。
- 技术成长空间大，可以从业务开发延展到架构、稳定性、工程效率、开源协作。
- 团队有技术分享和学习氛围，适合愿意持续提升的同学。

### 3. 候选人怎么准备

- Java、MVC、基础数据结构、操作系统、网络和数据库需要扎实。
- 面试不只看语言，也看学习能力、问题拆解能力和工程意识。
- 可以准备一个自己深入参与的项目，讲清背景、方案、取舍和结果。

## 表达注意点

- 直播时间短，要先讲最有吸引力的信息。
- 语速可以快，但关键卖点要停顿和重复。
- 少讲内部流程，多讲候选人关心的成长、机会、技术栈和岗位价值。
- 避免只罗列技术名词，最好用“能做什么、能成长什么、为什么重要”来表达。

## 后续复盘建议

| 维度 | 待补充问题 |
|---|---|
| 数据 | 直播观看人数、互动人数、简历投递数、有效候选人数。 |
| 渠道 | Boss 直聘直播与其他渠道相比的转化差异。 |
| 内容 | 哪些岗位卖点最能引发候选人兴趣。 |
| 流程 | 直播后如何承接候选人咨询、投递和面试。 |
